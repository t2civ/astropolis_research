# Background

I'm collecting descriptive information and input data for a simulation that focuses on resource streams: extraction, commodity trading, and downstream usage in industrial/economic activities. The simulation starts at present-day technology and operations on Earth, but runs forward in time to include highly speculative (yet physically plausible) future technologies and operations in the wider solar system. Abstractions, simplifications, approximations, and speculations are necessary. However, we want the overall simulation to adhere (at least approximately) to basic physical principles such as mass balance, energy conservation, etc.

# General Information

1. Descriptive information is provided in markdown (.md) files. Model input is formatted as data tables in tab-delimited (.tsv) files.
2. Markdown groupings (Energy, etc.) are for organization and display only; functional effects in the simulation are handled by individual item properties (although these coincide with groupings in some cases).
3. Data table details and instructions are in file _table_instructions.md; this should be provided if I'm asking for model input data.

3. Provide feedback and ask for clarifications that might help improve the prompt and/or context files for subsequent analyses.
4. Summarize simplifications and important omissions in your analyses.
