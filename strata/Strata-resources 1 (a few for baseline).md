## Key Simplifications & Omissions

**Composition fidelity.** "Stone" is defined as pyrolite (primitive mantle), which is Mg-rich ultramafic. This is a decent match for the Moon's mantle but a poor match for Earth's felsic upper continental crust (which is Si/Al/K-rich). Similarly, "Regolith" is basaltic—reasonable for the Moon but not for Earth soils. The model acknowledges this mismatch explicitly; within-resource composition is treated as fungible.

**Industrial minerals as catch-all.** The INDUSTRIAL_MINERALS category absorbs a very heterogeneous set of materials: limestone, clay, phosphate, salt, gypsum, aggregate, potash. On Earth this is ~5–10% of near-surface mass; the single compositional reference blurs major differences between carbonate platforms, evaporite basins, and clastic sedimentary sequences.

**Subsurface ore abundances are very small.** At 0.5–2 km depth the total stratum mass is ~22× the 0–100 m layer, so even if there is 2–3× more absolute ore at depth, the weight-percent drops by nearly an order of magnitude. Ores at 0.0001–0.005% may look surprisingly small, but high dispersion (1.2–2.0 in log₁₀) ensures that localized concentrations are orders of magnitude above the mean—matching real mining economics.

**Moon maria/terrae gap.** The anorthositic substrate beneath maria basalt (5–60 km depth, 17% area) is not explicitly modeled as a separate stratum. This creates a ~1% mass deficit. A simple fix would be to extend CRUST_TERRAE to area_fraction = 1 below 5 km and keep CRUST_MARIA as just the basaltic veneer.

**Territorial shelf apportionment is rough.** Continental shelf areas by territory are order-of-magnitude estimates. For the simulation these affect subsurface total mass per territory; actual EEZ shelf boundaries are geopolitically complex.

**No distinction between maria sub-types.** High-Ti vs. low-Ti mare basalt has a large effect on ilmenite abundance (5–25 vol% range), which is collapsed into a single CRUST_MARIA composition. Similarly, KREEP terrane is spatially concentrated around the Procellarum region but spread uniformly across terrae in this model.

## Feedback for Prompt / Context File Improvement

1. **Resource abundance magnitude guidance.** The most time-consuming and uncertain part was converting global ore reserve estimates into stratum weight-percent. A note in the extractable_resources file giving a rough expected range (e.g., "for Earth near-surface strata, ore resources are typically 10⁻⁵ to 10⁻¹ %wt with high dispersion") would greatly reduce ambiguity and improve consistency across analysts.

2. **Stone/Regolith composition vs. application.** These two filler resources have fixed compositions that don't match Earth's continental crust (too ultramafic) or most specific bodies. Consider either (a) adding a note that the compositions are irrelevant placeholders for the simulation (since downstream processes reference the resource name, not its elemental composition), or (b) defining location-specific variants (e.g., "Felsic Stone" for continental crust, "Mafic Stone" for oceanic/lunar crust). This would improve any downstream mass-balance check.

3. **Depth-band normalization.** The strata span very different thickness ranges (0.1 km surface vs. 1.5 km subsurface vs. 59 km crust). Since ores are a tiny fraction of total mass, their weight-percent varies enormously between thin and thick strata even when absolute tonnages are similar. Consider whether the model could alternatively accept absolute mass per resource per stratum (letting the code compute weight-percent from stratum total mass), which would be more intuitive for analysts.

4. **Maria/terrae boundary.** The current definition leaves the anorthositic substrate below maria basalt (5–60 km at 17% area) unassigned. I'd recommend either (a) defining CRUST_MARIA as the full crustal thickness in maria regions (~25 km, with a mixed basaltic-anorthositic composition), or (b) explicitly stating that CRUST_TERRAE extends under maria below the basaltic veneer (area_fraction 1 for depths >5 km).

5. **Territorial shelf data source.** If territorial subdivisions will be reused across multiple strata (shelf, near-surface, subsurface, deep subsurface, etc.), consider defining the territory-to-area mapping once in the strata file rather than leaving it implicit. This would prevent inconsistencies across runs.

6. **Table 2 format.** With 27 resource columns each containing 4 comma-separated values, the table is extremely wide (~750 characters per row). For production runs covering all ~100+ strata, consider a long-format table instead: `strata_name, resource_name, relative_abundance, abundance_cv, dispersion, dispersion_error` — one row per strata-resource pair. This is easier to validate, diff, and version-control, and zero entries can be omitted entirely.

7. **PRECIOUS_METAL_ORES and URANIUM_ORES on other bodies.** These are tagged "Earth only as main extraction target" but trace amounts exist in lunar KREEP and asteroidal material. Clarify whether the model should carry zero abundance off-Earth (with precious metals tracked only as IRON_NICKEL byproducts) or allow tiny non-zero values.
